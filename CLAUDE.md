# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal developer portfolio (single long-scroll landing page + a `/project` detail page) built with **TanStack Start** — a full-stack React 19 framework on Vite 7. SSR with prerendering, deployed to Netlify. Live at https://novodip.netlify.app/.

## Commands

```bash
npm run dev        # dev server on port 3000
npm run build      # production build (output: dist/client, per netlify.toml)
npm run preview    # preview the production build
npm run test       # vitest run (one-shot); use `npx vitest` for watch, `npx vitest <file>` for a single file
npm run lint       # eslint
npm run format     # prettier --check
npm run check      # prettier --write + eslint --fix (run before committing)
```

Note: there is no typecheck script — TS is `noEmit`. Run `npx tsc --noEmit` to typecheck.

## Architecture

**Routing is file-based** under `src/routes/`. TanStack Router generates `src/routeTree.gen.ts` automatically — **never edit it by hand**; it's regenerated by the dev server / router plugin.

- `__root.tsx` — the shell (`<html>`/`<body>`). Mounts `Header` and `ScrollProgress` **globally**, so section components must not render their own. Also injects a blocking inline `THEME_INIT_SCRIPT` in `<head>` to set the theme class before hydration (avoids FOUC).
- `index.tsx` — the home page: composes the section components (`HeroSection`, `AboutSection`, `ProjectsSection`, `ExperienceSection`, `SkillsSection`, `ContactSection`) in order.
- `project.tsx` — renders `components/ProjectPage.tsx`.
- `count.ts` — a **server route** (`server.handlers.GET`) that reads a resume-view counter from Supabase. Server-only handlers like this are how backend logic is exposed; keep server secrets out of client code.

**Content is data-driven.** Section copy lives in `src/data/*.json` (`hero`, `about`, `projects`, `skills`, `experience`, `contact`) and is imported directly into the matching section component. Prefer editing JSON over hardcoding copy — though note some visual detail (e.g. project screenshot layouts in `ProjectsSection.tsx`) is intentionally hardcoded in the component.

**Styling — Tailwind CSS v4** configured entirely in `src/styles.css` (no `tailwind.config`). Design tokens are HSL CSS variables under `@layer base` (`--background`, `--primary` = brand orange `#ff5c35`, etc.), mapped to utilities via `@theme inline`. Custom fonts: `font-display` (Bricolage Grotesque), `font-sans` (Space Grotesk), `font-mono` (JetBrains Mono). Custom animations (`animate-marquee`, `animate-float`, …) are also declared there.

**Theming** is a hand-rolled light/dark/auto system (not `next-themes` despite the dep). `ThemeToggle.tsx` cycles modes and writes `localStorage.theme` + a `data-theme` attr / `.light`/`.dark` class on `<html>`; the init script in `__root.tsx` must stay in sync with this logic.

**Animations** use the `motion` package (`motion/react`), not `framer-motion`. `components/Reveal.tsx` is the standard scroll-into-view fade/rise wrapper — reuse it for reveal effects.

**UI primitives** in `src/components/ui/` are shadcn/ui (style `radix-nova`, see `components.json`). Merge classes with `cn()` from `src/lib/utils.ts`.

**Integrations** (all via `VITE_*` env vars in `.env`, read through `import.meta.env`):
- `lib/supabase.ts` — Supabase client (analytics / resume-view count).
- `lib/sendEmail.ts` — EmailJS for the contact form; shows `sonner` toasts.
- `lib/seo.ts` — `injectHeadForSEO` returns meta/OG/Twitter tags + JSON-LD; wired into route `head` options.

## Conventions

- **Path aliases:** import from `#/...` (preferred) or `@/...`, both resolve to `src/`.
- **Prettier:** no semicolons, single quotes, trailing commas everywhere (`prettier.config.js`).
- ESLint extends `@tanstack/eslint-config`; import ordering/sorting rules are disabled.
